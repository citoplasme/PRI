Abstract

Este sexto trabalho semanal baseia-se na criação de um servidor em NodeJS com auxílio de ExpressJS referente à gestão de uma base de dados em MongoDB. Assim, com auxílio de roteadores e uma API de dados, sejam efetuadas operações de adição, remoção e listagem sobre a BD.

O servidor apenas poderia receber pedidos de GET para /, retornando uma página inicial; /filmes, listando todo os filmes existentes na BD; para /filmes/:id devendo apresentar um único filmes, representado pelo identificador em questão; e para /filmes/inserir, retornando uma página com um formulário. No que toca a pedidos de POST, caso fosse para /filmes, um novo filme deveria ser adicionado à BD. Note-se que, devido ao uso de um formulário estático, apenas existem cinco vagas quer no elenco, quer nos géneros do filme. Por fim, caso seja recebido um pedido de DELETE em /filmes/:id, o filme em questão deve ser removido da BD. é, ainda, de destacar que as operações de alteração sobre a BD passam pela API de dados, sendo que esta possui os seguintes pedidos: GET em /api/filmes e /api/filmes/:id; POST em /api/filmes; e DELETE em /api/filmes/:id.
